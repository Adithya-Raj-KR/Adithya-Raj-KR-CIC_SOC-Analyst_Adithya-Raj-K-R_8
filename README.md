---

## What is Incident Reporting?
Incident reporting is the process of documenting security incidents, including:
- What happened  
- When it happened  
- How it was detected  
- Actions taken  

It serves as an official record for investigation, analysis, and future prevention.

---

## Importance of Reporting
- Track security incidents  
- Improve detection and response  
- Provide evidence for investigations  
- Ensure communication across teams  

---

## Key Elements of an Incident Report
- Title  
- Date & Time  
- Affected System/User  
- Source IP  
- Description  
- Severity  
- Actions Taken  
- Status  

---

## Example Incident

**Title:** Brute Force Attack  
**Time:** Apr 10, 10:15 AM  
**User:** admin  
**IP:** 185.45.23.11  

**Description:**  
Multiple failed login attempts followed by a successful login, indicating a possible brute force attack and account compromise.

**Severity:** High  

**Actions Taken:**  
- Blocked IP  
- Reset password  
- Monitored system activity  

**Status:** Resolved  

---

## Severity Levels
- Low – Minor activity  
- Medium – Suspicious behavior  
- High – Confirmed attack  

---

## Best Practices
- Be clear and concise  
- Use log evidence  
- Avoid assumptions  
- Follow structured format  
- Maintain professional language  

---

## Conclusion
Incident reporting is a critical SOC skill that ensures proper documentation, effective communication, and improved security response.
